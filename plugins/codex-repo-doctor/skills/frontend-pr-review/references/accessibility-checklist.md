# Accessibility Checklist

- Interactive controls use native `button`, `a`, `input`, `select`, or `textarea` where possible.
- Inputs have visible labels or programmatic labels.
- Buttons expose text or accessible names.
- Images that convey meaning have useful `alt` text.
- Focus states remain visible.
- Keyboard navigation can reach and operate changed controls.
- ARIA is not used to replace native semantics unnecessarily.
- Error messages are associated with the affected controls.
